== Store Description (English) ==

Aegis — Your All-in-One Email & Browsing Shield

Tired of phishing emails and sketchy websites? Aegis protects you on two fronts: it guards your inbox and watches over every site you visit.

Smart Email Organizer — Open Gmail and Aegis instantly sorts your inbox into categories like Work, Shopping, Finance, and more. No more drowning in unread emails. Choose built-in rules for speed, or plug in your favorite AI model for even smarter sorting. Create your own categories with custom colors, icons, and keywords.

Email Safety Scanner — Click on any email and Aegis gives it a safety score from 0 to 100. It spots phishing tricks, flags suspicious links, and checks sender reputation — all in real time, all on your device. Your emails never leave your browser.

Domain Trust Checker — Visit any website and Aegis shows a color-coded trust score right on the extension icon: green means safe, amber means caution, red means danger. Click for the full story — when was this domain registered? Who owns it? Where's the server located? Brand-new domains and servers in risky regions get flagged automatically so you can browse with peace of mind.

Browsing Insights — See where your time goes online. Aegis categorizes your browsing across 20+ categories and shows beautiful charts of your daily habits, trends, and screen time. Export your data anytime.

Privacy First — Everything runs locally in your browser. No accounts, no cloud processing, no data collection. Your emails and browsing history stay yours.

== Single Purpose Description ==

Aegis helps users organize their Gmail inbox, detect email security threats, analyze website domain safety, and understand their browsing habits — all from a single extension. It categorizes emails using keyword rules or AI, scores email safety with phishing detection, evaluates domain trustworthiness via registration age and server geolocation, and tracks browsing activity across 20+ website categories with visual analytics.


== Permission Justifications ==

--- storage ---
Used to persist user settings (analysis mode, AI configuration, custom categories), EULA acceptance state, browsing history data, URL category mappings, and anonymous feedback preferences across sessions and devices via chrome.storage.sync and chrome.storage.local.

--- activeTab ---
Used to access the currently active tab's URL and title for URL categorization and browsing analytics tracking. This allows the extension to determine which website category the user is visiting without requiring broad host permissions for every page.

--- scripting ---
Used to inject the URL feedback widget into web pages when a user visits an uncategorized URL and has feedback mode enabled. The widget allows users to assign a category to the current site, improving the community URL category database.

--- alarms ---
Used to schedule periodic background tasks: weekly whitelist refresh, weekly URL category list sync from the server, daily cleanup of browsing history older than 30 days, and periodic flush of active browsing time tracking data (every 15 seconds).

--- webNavigation ---
Used to detect when a user completes navigating to a new page (top-level frame only). This triggers URL categorization, browsing time tracking for the analytics dashboard, domain security analysis (RDAP registration lookup and server geolocation scoring), and determines whether to show the feedback widget on uncategorized sites.

--- idle ---
Used to detect when the user is idle or has locked their screen, so the extension can pause active browsing time tracking. This ensures the analytics dashboard accurately reflects actual screen time rather than counting idle periods.

--- tabs ---
Used to get the URL and title of the active tab when tab focus changes or a tab updates. This is required for accurate browsing time tracking — the extension needs to know which tab is active to attribute time to the correct website category.

--- Host permissions (<all_urls>) ---
Required for multiple purposes: (1) content scripts inject into Gmail pages (mail.google.com) to provide email categorization and security analysis UI, (2) the webNavigation API needs host access to track page navigations across all sites for the browsing analytics feature, and (3) the domain security analyzer fetches registration data from rdap.org, resolves DNS via dns.google, and queries server geolocation from ip-api.com to score domain trustworthiness. The extension also makes API calls to its own backend (aegis.penrose.services) for syncing URL category lists and submitting anonymous feedback.


== Remote Code Declaration ==

Are you using remote code?
Yes

--- Reason ---
The extension connects to an OpenAI-compatible API endpoint (user-configured, default: api.openai.com) to perform AI-powered email categorization when the user enables AI mode in settings. The API key and endpoint URL are provided by the user in the extension settings page. The extension sends email metadata (subject, sender) to the configured API and receives a JSON classification response. No remote JavaScript code is loaded or executed — only REST API calls are made for data processing. Additionally, the extension fetches JSON data from its own backend API (aegis.penrose.services) to sync URL category lists and service whitelists. The domain security analyzer queries three external APIs for domain intelligence: rdap.org (WHOIS/RDAP registration data), dns.google (DNS-over-HTTPS resolution), and ip-api.com (IP geolocation). All responses are pure JSON data used for scoring — no executable code is loaded from these services. Results are cached locally for 7 days to minimize API calls.
