## Project Overview

**Aegis** is a Chrome Extension (Manifest V3) that injects into Gmail and provides:

- Right-side sliding panel: categorizes inbox emails with unread/all toggle
- Security analysis popup: auto-triggers when opening an email (safety score + issues + link audit)
- Dual analysis modes: local keyword rules (default, no API needed) or OpenAI-compatible AI
- Bulk email operations: checkbox select в†’ delete / move-to-label via simulated DOM events
- Domain whitelist validation: detects off-brand links and spoofing attempts
- No Gmail OAuth required вЂ” all operations through DOM event simulation

**Tech stack:** Vanilla JavaScript ES6+, Chrome MV3, Jest (unit tests), Makefile (build)

## Safety Score Logic (`email-analyzer.js: analyzeEmailDetail`)

Starts at 100. Deductions:

| Check | Deduction | Cap |
|---|---|---|
| No sender email address | -20 | вЂ” |
| Temp/disposable email service (from whitelist `suspiciousDomains`) | -30 | вЂ” |
| Local part has 4+ consecutive digits | -10 | вЂ” |
| Suspicious TLD (.xyz .top .click .loan .work .date .win .bid .stream) | -15 | вЂ” |
| Auto-generated local part (12+ alphanumeric chars, no symbols) | -5 | вЂ” |
| Phishing keywords in subject+body | -10/each | -30 |
| HTTP links (not HTTPS) | -5/each | -15 |
| Potential spoofing (keyword match but sender domain doesn't match service) | -25/each | -40 |
| Off-whitelist links (sender matches service but link domain doesn't) | -15/each | -30 |
| Suspicious domain links (IP address, suspicious TLD) | -15/each | -30 |

| Score | Level | Color |
|---|---|---|
| 80вЂ“100 | safe | `#1a7f37` |
| 50вЂ“79 | caution | `#9a6700` |
| 0вЂ“49 | danger | `#cf222e` |

## Whitelist System (`src/data/service-whitelist.json`)

Schema:
```json
{
  "version": "2",
  "updatedAt": "...",
  "services": [
    {
      "id": "google",
      "name": "Google",
      "keywords": ["google", "gmail"],
      "baseDomains": ["google.com", "gmail.com"],
      "senderDomains": ["google.com", "gmail.com"]
    }
  ],
  "publicEmailDomains": ["gmail.com", "yahoo.com", ...],
  "suspiciousDomains": ["tempmail.com", "guerrillamail.com", ...],
  "shortUrlServices": ["bit.ly", "tinyurl.com", ...]
}
```

**WhitelistManager methods:**
- `init()` вЂ” load from background cache (call on content script start)
- `findServiceBySenderDomain(senderEmail)` вЂ” match sender's domain to a service
- `findServicesByKeywords(emailContent)` вЂ” keyword-based service detection (for spoofing)
- `isDomainInService(hostname, service)` вЂ” check if link belongs to a service's baseDomains
- `isKnownShortUrl(url)` вЂ” check if URL uses a known shortener
- `isPublicEmailDomain(domain)` вЂ” detect personal email services
- `isSuspiciousDomain(domain)` вЂ” detect temp/disposable email services
- `extractBaseDomain(hostname)` вЂ” handles .co.jp, .com.tw special TLDs

**Domain matching rules:**
- Base domain matching: `mail.google.com` в†’ `google.com`
- Special TLD handling: `service.ntpc.gov.tw` matches `gov.tw`
- Subdomain matching: `subdomain.example.com` matches `example.com`

**First install:**
```

  в†’ background.js reads chrome.storage.sync
  в†’ writes DEFAULT_SETTINGS for any missing keys
  в†’ seeds bundled whitelist to chrome.storage.local

**User opens Gmail (new tab):**
```

  в†’ checks window.__aegisInitialized (prevents duplicate injection)
  в†’ new GmailPlatform()

  в†’ WhitelistManager.init() loads whitelist
  в†’ platform.observeNavigate() watches SPA route changes

```

  в†’ sendMessage ANALYZE в†’ content.js: runAnalysis()
```

## AI Mode

**Single email analysis (`AI_ANALYZE`):**
- Background fetches `{aiSettings.baseUrl}/chat/completions`
- System prompt: classify into one category from the provided list
- User message: Subject + From + Body[:1000] + Links[:10]
- Returns `{ category: "category name" }`

**Batch analysis (`AI_BATCH_ANALYZE`):**
- Sends array of `{ id, subject, sender, senderEmail }`
- Returns `{ results: [{ id, category }] }`

**Model list fetching (`FETCH_AI_MODELS`):**
- OpenAI-compatible: `GET /models` with Authorization header
- Gemini: `GET /v1beta/models?key=API_KEY`

```javascript

  name: 'ж—…йЃЉ',

```

Users can also add/edit/remove categories via the options page.

### Add a New Platform (Yahoo Mail, Outlook)

1. Create `src/platforms/yahoo-platform.js` extending `BasePlatform`
2. Implement all abstract methods: `getName()`, `isMatchingPage()`, `getEmails()`, `getLabels()`, `getEmailDetail()`, `deleteEmails()`, `moveToLabel()`, `observeNavigate()`
3. Add URL pattern to `manifest.json` в†’ `content_scripts[0].matches`
4. Add `src/platforms/yahoo-platform.js` to `manifest.json` content_scripts load order (before `content.js`)
5. In `content.js`, select platform by URL:
```javascript
const platform = window.location.href.includes('yahoo.com')
  ? new YahooPlatform()
  : new GmailPlatform();
```

### Add Phishing Keywords

Edit `PHISHING_KEYWORDS` array at top of `src/analysis/email-analyzer.js`.

### Add Whitelist Services

Add entries to `src/data/service-whitelist.json` в†’ `services[]`. Each service needs `id`, `name`, `keywords[]`, `baseDomains[]`, `senderDomains[]`.

### Add Gmail Locale Support

In `gmail-platform.js` в†’ `GMAIL_ACTIONS`, add the locale's tooltip strings to `tooltips[]` and `ariaLabels[]`. For substring matching, add to `tooltipContains[]`.

## Known Limitations

### Gmail DOM Fragility
Gmail is a complex SPA; DOM operations are unsupported and may fail:
- **Checkbox not responding**: `_selectRows()` dispatches `mouseenter`/`mouseover` to force checkbox render, but Gmail's virtual DOM may still not render it
- **Toolbar delay**: `_waitForElement()` polls for 3s max, then falls back to keyboard shortcut `#` (Shift+3) for delete
- **Class name changes**: update `GMAIL_ACTIONS` and `getEmails()` selectors if Gmail updates break functionality

### Content Script Injection Timing
If Gmail was open before the extension was installed, Chrome won't auto-inject. `popup.js` в†’ `ensureContentScript()` handles this via dynamic injection fallback.

### CORS for AI API
AI API calls must go through `background.js` (service worker). Direct fetch from content script will be blocked by CORS.

### Storage Quota
`chrome.storage.sync`: 8KB per item, 100KB total. Heavy keyword usage may need migration to `chrome.storage.local` (5MB limit).

### Gmail SPA Navigation
URL hash changes are monitored via `MutationObserver` on `document.title` + `hashchange` events. Email detail pages are detected by regex `/^#[^/]+\/[A-Za-z0-9]{10,}/`.

## Code Conventions

- **Classes**: PascalCase (`AnalysisPanel`, `GmailPlatform`)
- **Functions**: camelCase (`analyzeEmailList`, `getSettings`)
- **Constants**: UPPER_SNAKE_CASE (`PHISHING_KEYWORDS`, `SUSPICIOUS_TLDS`)
- **CSS**: kebab-case with `aegis-` prefix (`.aegis-panel`, `#aegis-email-popup`)
- **All modules**: IIFE pattern with `window.ModuleName` export
- **DOM selectors**: prefer `[role="..."]` attributes over class names for stability
- **Error handling**: try-catch all async ops; fall back to local analysis if AI fails; `console.warn` for non-critical errors; alert only for actionable failures
